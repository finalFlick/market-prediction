# FEATURE-0028: Run, Strategy, and Config API Surfaces

## Parent Epic

backend-api-streaming

## Status

Proposed

## Summary

Define API routes and contracts that expose run state, strategy metadata, configuration, and audit views to the dashboard.

## Why

The dashboard needs typed backend contracts rather than direct database coupling.

## What

A read-first API capability with route scope, response contracts, error handling, and phase-specific write boundaries.

## User / System Story

As a platform operator or implementation agent,
I want run, strategy, and config api surfaces,
so that the trading-lab platform can satisfy the related requirements with evidence.
